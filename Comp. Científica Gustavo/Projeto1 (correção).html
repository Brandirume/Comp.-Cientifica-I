def biss_solve(f, y, a, b, xtol=1e-8, ytol=1e-8):
    """Bissects the interval  [a,b]  until an approximate solution to  f(x) = y  is found.
    Also returns the number of bissections"""
    eq = lambda x: f(x) - y
    m = (a+b)/2
    nbiss = 1
    if eq(a)==0:
        return a, nbiss
    elif eq(b) == 0:
        return b, nbiss
    while abs(b-a)>xtol and abs(eq(m))>ytol:
        eqm = eq(m)
        if eqm*eq(a) < 0:
            b = m
            m = (a+b)/2 
        elif eqm*eq(b) < 0:
            a = m
            m = (a+b)/2 
        elif eqm==0:
            return m, nbiss
        else:
            return "Sem soluo", nbiss
        nbiss+=1
    return m, nbiss

def newton_solve(f, df, y, x0, xtol=1e-8, ytol=1e-8, maxiter=100):
    """Newton's algorithm for solving  f(x) = y  starting from  x0.
    Also returns the number of evaluations of f"""
    nevals = 0
    x = x0
    def g(x): return f(x) - y
    
    for i in range(0,maxiter):
        g1 = g(x)
        step = g1 / df(x)
        nevals += 1
        newx = x - step
        if abs(step) < xtol or abs(g1) < ytol:
            return newx,nevals
        x = newx
    return newx,nevals

def newton_inv(f, df, ys, x0, xtol=1e-8, ytol=1e-8, maxiter=100):
    """Newton's algorithm for solving  f(x) = y  for each  y  in the list  ys.
    Returns two lists, one of the (approximate) roots, another of the iterations for each one."""
    xs     = []
    nevals = []
    
    for i in range(len(ys)):
        x, n = newton_solve(f, df, ys[i], x0, xtol=xtol, ytol=ytol)
        nevals.append(n)
        xs.append(x)
        x0 = x
    return xs,nevals

A aproximao sucessiva do mtodo de newton utiliza a ltima raiz como valor inicial para encontrar a prxima raz para um outro valor de $y$. Na bisseo, podemos encontrar a raz para vrios valores de $y$, mas no h o argumento de valor inicial, $x0$. Em vez disso, h um intervalo a ser determinado. Poderamos atualizar $a$ ou $b$ do intervalo passado, conforme encontramos novas razes, mas qual dos dois atualizar com o valor da raz encontrada vai depender de como variamos o $y$ a ser encontrado. Considerando a questo 6, com $f(x) = sen(x)$, e $y$ indo de $0$ a $0.9$, podemos diminuir o valor do extremo final do intervalo - ou seja, $b$ - passado para a bisseo, para ser o valor da ltima raz encontrada, j que o prximo $y$, que maior, vai estar em um $x$ anterior raz encontrada para tal $y$ considerado anteriormente, j que a funo decrescente em $x \in [0,\pi/2]$, ou seja, valores maiores de y, esto em menos valores x. Se a funo fosse crescente, para cada $y$ maior, o $x$ tambm seria maior, ento teramos que atualizar o extremo inferior, $a$, com a raz encontrada anteriormente. Isto tudo quer dizer que precisamos saber se a funo cresce ou decresce a cada novo y a ser calculado.
